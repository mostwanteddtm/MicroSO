There is no DOS in either W2K or XP. When you open up a command window that isn't actually DOS, just a command line interpreter for XP. There also isn't a setting in the compatibility wizard for DOS programs, the earliest OS being Win95, so you can't run pure DOS programs in XP. This is by design from Microsoft, and is to make the OS more stable.

The thing to be aware of if you load parameters through autoexec.nt or config.nt for DOS programs is that what you see in the Window is not a pure DOS environment, but a Virtual DOS Machine (an NTVDM).

If your program requires a pure DOS environment, such as direct access to hardware or similar, it will not run in an NTVDM as this is just an emulation of DOS.
